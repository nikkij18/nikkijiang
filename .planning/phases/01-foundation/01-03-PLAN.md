    ```tsx
    import { getAllProjects } from '@/lib/projects'

    export async function generateStaticParams() {
      const projects = getAllProjects()
      return projects.map((project) => ({
        slug: project.slug,
      }))
    }

    export default async function ProjectPage({
      params,
    }: {
      params: Promise<{ slug: string }>
    }) {
      const { slug } = await params
      return (
        <main className="min-h-screen flex flex-col items-center justify-center gap-4">

        </main>
      )
    }
    ```

<threat_model>
## Trust Boundaries

Static, public Next.js portfolio. The dynamic route `[slug]` reads from a build-time TypeScript constant (no runtime data fetch), so the only "input" is the URL path itself, which Next.js validates against the `generateStaticParams` allow-list.

| Boundary | Description |
|----------|-------------|
| URL path → `[slug]` route | Next.js matches against the prerender list; non-matching slugs return 404 by default (`dynamicParams` defaults differ by Next.js version — verify in node_modules/next/dist/docs/) |
| Build → static HTML | All five slug pages are emitted as static HTML at `next build` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | T (Tampering — XSS via slug interpolation) | app/projects/[slug]/page.tsx | mitigate | The slug is rendered as a React text node (`{slug} — coming soon.`), which auto-escapes. No `dangerouslySetInnerHTML`, no innerHTML write. Acceptance criteria forbid these. Even if a malicious slug were requested, Next.js would 404 because the slug is not in the `generateStaticParams` list. |
| T-01-03-02 | S (Spoofing — unauthorized slug) | app/projects/[slug]/page.tsx | mitigate | `generateStaticParams` returns exactly the 5 known slugs. Default Next.js behavior is to 404 unknown slugs at runtime. We do not set `dynamicParams = true`, so no on-demand rendering of unknown slugs. |
| T-01-03-03 | I (Information disclosure — debug leak) | stub pages | mitigate | Stub copy is hand-authored and contains no PII or secrets ("Coming soon.", "{slug} — coming soon."). Reviewer: do not introduce `console.log(process.env)` or any env-var read in stub files. |
| T-01-03-04 | D (Denial of service — build size) | scaffold SVG cleanup | accept | LOW. Deleting 5 SVGs reduces the static bundle marginally; no DOS surface either way for a portfolio. |
| T-01-03-05 | T (Tampering — supply chain on path alias) | @/lib/projects import | accept | The `@/*` alias resolves to the local working directory (`tsconfig.json`). No third-party module is shadowed by the alias because `lib/` is project-owned. No action needed. |

Static-site posture: ASVS L1 surface is satisfied by React's default escaping plus the closed-set `generateStaticParams` allow-list. No high-severity threats introduced by this plan.
</threat_model>

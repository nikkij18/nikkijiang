## Trust Boundaries

Static, public, read-only Next.js portfolio. No authentication, no user input, no API surface, no database. The only outbound interactions are link navigation and image loading from the local `/public` folder.

| Boundary | Description |
|----------|-------------|
| client browser → static HTML | Pre-rendered HTML and JS shipped from Vercel |
| meta description → search engines | Indexable metadata visible in search results and social previews |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | I (Information disclosure) | app/layout.tsx metadata | mitigate | Description is hand-authored marketing copy. No PII beyond first/last name (already public). No secrets, no env vars referenced. Reviewer: do not introduce `process.env.*` reads in metadata. |
| T-01-02-02 | T (Tampering — XSS) | components/Hero.tsx, components/NavBar.tsx, app/layout.tsx | mitigate | All copy is rendered as React text nodes (auto-escaped). No `dangerouslySetInnerHTML`, no innerHTML assignment, no untrusted-source interpolation. Acceptance criteria forbid adding these. |

STATE.md tracks the file as a content blocker, not a security control. |
